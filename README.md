# تطبيق أثر (Athar Mobile App) — دليل التطوير والبناء والنشر

مستودع تطبيقات الجوال الخاص بمشروع **أثر (Athar)**، وهو تطبيق هجين مبني باستخدام إطار العمل الحديث **Expo 54+** (React Native) و**TypeScript** مع هيكلة إدارة متكاملة ومستقرة للأمان والأداء.

---

## 🛠️ المتطلبات التقنية الأساسية
قبل البدء بالعمل على التطبيق، يرجى التأكد من تهيئة بيئة العمل المحلية لديك بالمتطلبات التالية:
* **Node.js**: الإصدار `20` أو أحدث.
* **إدارة الحزم**: استخدام `pnpm` حصرياً (Pnpm Workspace).
* **بيئة الأندرويد**: Android Studio مع تثبيت Android SDK و Emulator مهيأ للتشغيل.
* **بيئة الآيفون**: نظام macOS مع Xcode (الإصدار الاستقرائي الأخير) وتثبيت CocoaPods.

---

## 🚀 التشغيل والتطوير المحلي
لتشغيل مشروع الجوال محلياً والبدء بالتطوير السريع:

1. **تثبيت الاعتماديات والمكتبات:**
   ```bash
   pnpm install
   ```

2. **تهيئة ملف البيئة (.env):**
   تأكد من وجود ملف `.env` في جذر مجلد `mobile/` ويحتوي على رابط الخادم:
   ```env
   EXPO_PUBLIC_API_URL=https://athar.elcomlab.site/api
   ```

3. **بدء تشغيل خادم التطوير (Metro Bundler):**
   ```bash
   pnpm start
   ```

## 🤖 دليل بناء ونشر نسخة الأندرويد (Android Build & Google Play)

تتوفر طريقتان متكاملتان لبناء تطبيق الأندرويد:

### 1. البناء المحلي بالكامل (Local Build)
تعتمد هذه الطريقة على جهازك المحلي لإنتاج ملف APK مباشر للتجربة:

* **توليد المجلد الأصلي للأندرويد واستيراد المكتبات (Prebuild):**
  ```bash
  npx expo prebuild --platform android
  ```

* **التشغيل المباشر للتطوير على هاتف حقيقي أو محاكي:**
  ```bash
  npx expo run:android
  ```

* **بناء نسخة APK نهائية ومستقلة بالكامل (Release APK):**
  لتصدير ملف التثبيت المباشر دون الحاجة للسيرفر المحلي:
  ```bash
  cd android && ./gradlew assembleRelease
  ```
  > **ملاحظة:** ستجد الملف المترجم APK جاهزاً للتثبيت في المسار التالي:
  > `android/app/build/outputs/apk/release/app-release.apk`

---

### 2. البناء والنشر للمتجر عبر السحاب (Expo Application Services - EAS)
الطريقة الرسمية والمعتمدة لبناء التطبيقات سحابياً ورفعها لمتجر **Google Play Console**:

* **تثبيت أداة EAS وتدشين الدخول:**
  ```bash
  pnpm add -g eas-cli
  eas login
  ```

* **تهيئة إعدادات البناء السحابي (أول مرة فقط):**
  ```bash
  eas build:configure
  ```

* **بناء نسخة APK تجريبية (Preview) للتحميل الفوري عبر رمز الاستجابة السريعة:**
  ```bash
  eas build --platform android --profile preview
  ```

* **بناء النسخة الإنتاجية الرسمية بصيغة AAB (Android App Bundle) لمتجر جوجل بلاي:**
  ```bash
  eas build --platform android --profile production
  ```
  بعد اكتمال البناء السحابي، يمكنك تحميل ملف `.aab` وتوجيهه لمتجر **Google Play Console** تحت مسار الإنتاج أو الفحص المغلق.

## 🍎 دليل بناء ونشر نسخة الآيفون (iOS Build & App Store/TestFlight)

يتم بناء ونشر تطبيق الآيفون آلياً بالكامل عبر نظام **GitHub Actions CI/CD** الفاخر المدمج بالمشروع لضمان أسرع تجميع وأمان عالي:

### 1. البناء المحلي والتشغيل (Local Development Build)
* **توليد مجلد iOS وتحديث CocoaPods:**
  ```bash
  npx expo prebuild --platform ios
  cd ios && pod install
  ```

* **تشغيل التطبيق على محاكي الآيفون أو جهاز حقيقي موصول:**
  ```bash
  npx expo run:ios
  ```

---

### 2. النشر التلقائي لمتجر App Store / TestFlight
لقد قمنا ببرمجة خطاف النشر المؤتمت بالكامل بمجرد رفع التحديثات إلى فرع الإنتاج:

* **طريقة تفعيل البناء التلقائي:**
  كل ما عليك فعله هو دفع الأكواد إلى الفرع الرئيسي `main` في مستودع Git:
  ```bash
  git add .
  git commit -m "feat: update mobile app features and bump version"
  git push origin main
  ```

* **ماذا يحدث خلف الكواليس في خوادم GitHub؟**
  1. تقوم أداة الـ CI بسحب الكود بأحدث إصدار.
  2. يتم تهيئة شهادة آبل للتوزيع الرقمي (Apple Distribution Certificate) وملف التعريف البرمجي (Provisioning Profile).
  3. يتم تفعيل `expo prebuild` وتثبيت الكوكوبودز.
  4. يتم استدعاء `xcodebuild` لتجميع مساحة العمل المعتمدة `Athar.xcworkspace` والمخطط `Athar`.
  5. يتم ضغط التطبيق بصيغة `.ipa` وتصديره ورفعه تلقائياً ومباشرة إلى حسابك على **App Store Connect / TestFlight** ليكون متاحاً للمفحوصين فوراً!

## 🔒 الميزات المتقدمة والأمنية في التطبيق

### 1. نظام التحديث الإجباري الذكي (Force Update)
يمتلك التطبيق معالجاً ذاتياً متطوراً يتصل بالخادم ويقارن إصدار التطبيق المحلي الحالي مع أدنى إصدار مطلوب على السيرفر بشكل صامت. في حال تطلب البناء تحديثاً، يقفل التطبيق الشاشة بواجهة زجاجية راقية توجّه المستخدم مباشرة لصفحة التحميل المناسبة بالمتجر لمنع أي تعارض برمجيات مع السيرفر.

### 2. تشفير الكاش المحلي الفائق (Hermes & Storage Security)
جميع تداولات الذاكرة المؤقتة والمنشورات المخزنة محلياً يتم تشفيرها وفك تشفيرها ديناميكياً باستخدام تشفير **AES-256** الفاخر لحماية خصوصية المستخدمين بالكامل. يحتوي النظام على محول أوتوماتيكي ذكي (In-Memory Fallback) لضمان عدم تعطل التطبيق أبداً حتى في بيئات التطوير التي يغيب فيها الربط الأصلي للتخزين.
